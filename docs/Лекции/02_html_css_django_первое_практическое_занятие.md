## 1. HTML

HTML расшифровывается как HyperText Markup Language, язык гипертекстовой разметки.

HTML используется для описания структуры документа, который отображается браузером. Это не язык программирования, а язык разметки.

Главные свойства HTML:

- переносимость между устройствами и браузерами;
- простота синтаксиса;
- древовидная структура документа;
- устойчивость к ошибкам разметки.

Современный стандарт HTML развивается как living standard: изменения добавляются постепенно, а не только через крупные версии.

## 2. Минимальный HTML-документ

```html
<!doctype html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Моя страница</title>
  </head>
  <body>
    <h1>Привет, мир!</h1>
    <p>Это первый HTML-документ.</p>
  </body>
</html>
```

Назначение основных частей:

- `<!doctype html>`: сообщает браузеру, что используется HTML5.
- `html`: корневой элемент документа.
- `head`: служебная информация, стили, скрипты, метаданные.
- `body`: видимое содержимое страницы.
- `title`: заголовок вкладки браузера.
- `meta viewport`: корректное отображение на мобильных устройствах.

## 3. HTML не является строгим языком

Браузеры стараются исправлять ошибки в HTML:

- могут автоматически закрывать теги;
- могут добавлять отсутствующие `html`, `head`, `body`;
- могут исправлять неправильную вложенность;
- игнорируют часть ошибок.

Но полагаться на это нельзя. Хорошая практика:

- писать теги в нижнем регистре;
- закрывать парные теги;
- не делать перекрестную вложенность;
- заключать значения атрибутов в кавычки;
- использовать валидную структуру документа.

## 4. Блочные и строчные элементы

### Блочные элементы

Блочные элементы обычно занимают всю доступную ширину и начинаются с новой строки.

Примеры:

- `h1`-`h6`: заголовки;
- `p`: абзац;
- `div`: универсальный блочный контейнер;
- `hr`: горизонтальная линия;
- `pre`: предварительно форматированный текст;
- `ul`, `ol`, `li`: списки;
- `table`: таблица.

### Строчные элементы

Строчные элементы располагаются внутри строки и занимают только нужную ширину.

Примеры:

- `a`: ссылка;
- `span`: универсальный строчный контейнер;
- `strong`: важное выделение;
- `b`: визуальное жирное выделение;
- `em`: смысловой акцент;
- `i`: визуальный курсив;
- `img`: изображение.

Разделение на блочные и строчные элементы можно менять через CSS-свойство `display`.

## 5. Ссылки, изображения и якоря

Ссылка:

```html
<a href="https://example.com">Перейти на сайт</a>
```

Открытие в новой вкладке:

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">Открыть</a>
```

Изображение:

```html
<img src="image.jpg" alt="Описание изображения">
```

Якорная ссылка:

```html
<a href="#section-1">К разделу</a>

<h2 id="section-1">Раздел 1</h2>
```

Фрагмент после `#` обрабатывается браузером и не отправляется на сервер.

## 6. Формы

Формы используются для отправки пользовательских данных на сервер.

```html
<form method="post" action="/feedback/">
  <label>
    Имя:
    <input type="text" name="name">
  </label>

  <label>
    Сообщение:
    <textarea name="message"></textarea>
  </label>

  <button type="submit">Отправить</button>
</form>
```

Важные атрибуты:

- `method`: HTTP-метод, обычно `get` или `post`.
- `action`: URL, куда отправляются данные.
- `name`: имя поля, под которым значение уйдет на сервер.
- `type`: тип input, например `text`, `password`, `number`, `file`, `checkbox`, `radio`.
- `value`: значение по умолчанию.

Если `method` не указан, форма по умолчанию отправляет `GET`-запрос.

## 7. CSS

CSS расшифровывается как Cascading Style Sheets, каскадные таблицы стилей.

CSS отвечает за внешний вид HTML-документа: размеры, цвета, шрифты, отступы, расположение элементов.

Способы подключения CSS:

Внешний файл:

```html
<link rel="stylesheet" href="style.css">
```

Внутри документа:

```html
<style>
  p {
    color: green;
  }
</style>
```

Встроенный стиль:

```html
<p style="color: green;">Текст</p>
```

Лучше использовать внешние CSS-файлы: они переиспользуются, кешируются браузером и не смешивают структуру с оформлением.

## 8. CSS-селекторы

Основные селекторы:

```css
* { box-sizing: border-box; }

p { color: black; }

.btn { color: white; }

#main-title { font-size: 32px; }
```

Сложные селекторы:

```css
div.article a { color: blue; }

article > img { max-width: 100%; }

h1, h2, h3 { color: red; }

button.primary { background: royalblue; }
```

Важно:

- `div.article` означает `div` с классом `article`.
- `div .article` означает элемент с классом `article` внутри `div`.
- `>` выбирает непосредственного потомка.
- `,` группирует несколько селекторов.

## 9. Псевдоклассы

Псевдоклассы задают стили для состояния элемента.

```css
a:visited { color: purple; }
a:hover { color: red; }
input:focus { outline: 2px solid blue; }
li:first-child { font-weight: bold; }
```

Частый пример для интерактивных элементов:

```css
button:hover {
  cursor: pointer;
  background-color: #0d6efd;
}
```

## 10. Специфичность CSS

Если несколько правил подходят к одному элементу, браузер выбирает более специфичное.

Упрощенная система баллов:

- inline-стиль: 1000;
- `id`: 100;
- класс, атрибут, псевдокласс: 10;
- тег и псевдоэлемент: 1.

Пример:

```css
div { color: black; }          /* 1 */
.box { color: green; }         /* 10 */
div.box { color: red; }        /* 11 */
#main-box { color: blue; }     /* 100 */
```

Если специфичность одинаковая, применяется правило, которое написано позже.

`!important` перебивает обычный каскад, но использовать его нужно редко и осторожно.

## 11. Display и Visibility

```css
.hidden-display {
  display: none;
}

.hidden-visibility {
  visibility: hidden;
}
```

Разница:

- `display: none`: элемент не отображается и не занимает место.
- `visibility: hidden`: элемент не виден, но место под него сохраняется.

## 12. Позиционирование

Основные значения `position`:

- `static`: обычное положение элемента.
- `relative`: смещение относительно исходного места.
- `absolute`: позиционирование относительно ближайшего позиционированного родителя или документа.
- `fixed`: позиционирование относительно окна браузера.

Пример фиксированной кнопки:

```css
.help-button {
  position: fixed;
  right: 24px;
  bottom: 24px;
}
```

## 13. Box Model

Каждый элемент состоит из:

- `content`: содержимое;
- `padding`: внутренний отступ;
- `border`: граница;
- `margin`: внешний отступ.

Рекомендуемая базовая настройка:

```css
* {
  box-sizing: border-box;
}
```

Так размеры элементов считаются предсказуемее: `width` включает content, padding и border.

## 14. Адаптивная верстка

Адаптивная верстка позволяет странице хорошо отображаться на разных экранах: телефонах, планшетах, ноутбуках и больших мониторах.

Инструменты:

- относительные единицы: `%`, `vw`, `vh`, `rem`;
- `max-width` для ограничения ширины контента;
- media queries;
- flexbox и grid;
- CSS-фреймворки.

Пример media query:

```css
.layout {
  display: grid;
  grid-template-columns: 1fr 1fr;
}

@media (max-width: 768px) {
  .layout {
    grid-template-columns: 1fr;
  }
}
```

## 15. Bootstrap

Bootstrap — CSS-фреймворк, который предоставляет готовые компоненты и адаптивную сетку.

Подключение через CDN:

```html
<link
  href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css"
  rel="stylesheet"
>
<script
  src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"
></script>
```

Кнопка:

```html
<button class="btn btn-primary">Сохранить</button>
```

Сетка Bootstrap основана на 12 колонках. Число 12 удобно, потому что делится на 1, 2, 3, 4 и 6.

Пример:

```html
<div class="container">
  <div class="row">
    <div class="col-12 col-lg-6">Левая колонка</div>
    <div class="col-12 col-lg-6">Правая колонка</div>
  </div>
</div>
```

На маленьких экранах каждая колонка занимает всю ширину, на больших — половину.

## 16. Виртуальные окружения Python

Проблема: разные проекты могут требовать разные версии Django и других библиотек.

Если устанавливать все глобально через `pip install`, зависимости разных проектов будут конфликтовать.

Решение: виртуальное окружение. Для каждого проекта создается отдельная папка `.venv`, где лежат собственные зависимости.

Классический способ:

```bash
python -m venv .venv
source .venv/bin/activate
pip install django
```

На Windows активация выполняется другим скриптом из `.venv\Scripts`.

## 17. uv

`uv` автоматизирует работу с окружениями и зависимостями.

Создание проекта:

```bash
uv init
```

Добавление Django:

```bash
uv add django
```

Запуск команды внутри окружения:

```bash
uv run django-admin --version
```

Преимущества `uv`:

- быстро устанавливает зависимости;
- создает и использует виртуальное окружение автоматически;
- фиксирует зависимости в `pyproject.toml` и lock-файле;
- может управлять версией Python;
- упрощает запуск команд без ручной активации `.venv`.

В `.gitignore` обычно добавляют:

```gitignore
.venv/
```

## 18. Создание Django-проекта

Создание проекта:

```bash
uv run django-admin startproject awesome_site
```

Структура:

```text
awesome_site/
  manage.py
  awesome_site/
    __init__.py
    settings.py
    urls.py
    asgi.py
    wsgi.py
```

Назначение файлов:

- `manage.py`: утилита управления проектом.
- `settings.py`: настройки Django-проекта.
- `urls.py`: маршруты проекта.
- `asgi.py`, `wsgi.py`: точки входа для серверов приложения.

Запуск сервера разработки:

```bash
cd awesome_site
uv run python manage.py runserver
```

По умолчанию сервер доступен на:

```text
http://127.0.0.1:8000/
```

## 19. Миграции и база данных

Django по умолчанию использует SQLite. После создания проекта Django предупреждает о непримененных миграциях.

Применение миграций:

```bash
uv run python manage.py migrate
```

Миграции создают таблицы для встроенных приложений Django:

- пользователи;
- группы и права;
- админка;
- сессии;
- content types.

Миграция — это набор инструкций, который приводит структуру базы данных к нужному состоянию.

## 22. Первая модель

Пример модели сообщения:

```python
from django.db import models


class ChatMessage(models.Model):
    text = models.CharField(max_length=255)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"{self.id}: {self.text}"
```

Создание миграции:

```bash
uv run python manage.py makemigrations
```

Применение миграции:

```bash
uv run python manage.py migrate
```

Django автоматически добавляет поле `id` как первичный ключ, если не указать другой primary key.

## 23. Регистрация модели в админке

`chat_app/admin.py`:

```python
from django.contrib import admin

from .models import ChatMessage


@admin.register(ChatMessage)
class ChatMessageAdmin(admin.ModelAdmin):
    pass
```

После этого модель появится в админке, и через интерфейс можно будет создавать, редактировать и удалять сообщения.

## 24. Связи между моделями

Пример профиля пользователя и связи сообщения с автором:

```python
from django.contrib.auth.models import User
from django.db import models


class Profile(models.Model):
    user = models.ForeignKey(User, on_delete=models.PROTECT)
    name = models.CharField(max_length=255)

    def __str__(self):
        return self.name


class ChatMessage(models.Model):
    author = models.ForeignKey(Profile, on_delete=models.PROTECT)
    text = models.CharField(max_length=255)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"{self.id}: {self.text}"
```

`on_delete=models.PROTECT` запрещает удалить профиль, если на него ссылаются сообщения.

Важно: не нужно без необходимости делать `null=True`. Если поле логически обязательно, оно должно быть обязательным и в базе данных.

## 25. View и URL

View в Django — это функция или класс, который принимает HTTP-запрос и возвращает HTTP-ответ.

Простой view:

```python
from datetime import datetime

from django.http import HttpResponse


def chat_view(request):
    return HttpResponse(f"Привет, {request.user}! Сейчас {datetime.now()}")
```

Маршрут в `urls.py`:

```python
from django.contrib import admin
from django.urls import path

from chat_app.views import chat_view

urlpatterns = [
    path("admin/", admin.site.urls),
    path("chat/", chat_view),
]
```

После этого страница будет доступна по адресу:

```text
http://127.0.0.1:8000/chat/
```

## 26. Почему не нужно собирать HTML строками

Технически можно вернуть HTML прямо из `HttpResponse`, но так делать не стоит.

Плохой подход:

```python
def page(request):
    html = "<html><body><h1>Привет</h1></body></html>"
    return HttpResponse(html)
```

Проблемы:

- смешиваются HTML и Python-код;
- сложно поддерживать;
- легко допустить ошибку в разметке;
- повышается риск инъекций;
- нельзя нормально переиспользовать шаблоны.

Правильный подход — использовать шаблонизатор:

```python
from django.shortcuts import render


def page(request):
    return render(request, "page.html", {"title": "Привет"})
```

Шаблоны будут подробно разбираться отдельно.

## Термины

- HTML: язык гипертекстовой разметки.
- CSS: язык описания внешнего вида документа.
- Селектор: правило выбора HTML-элементов для применения CSS.
- Специфичность: приоритет CSS-правила.
- Bootstrap: CSS-фреймворк с готовыми компонентами и сеткой.
- Виртуальное окружение: изолированная среда зависимостей Python-проекта.
- `uv`: инструмент управления Python-проектами и зависимостями.
- Django-проект: конфигурация всего сайта.
- Django-приложение: отдельный модуль внутри проекта.
- Модель: Python-класс, описывающий таблицу базы данных.
- Миграция: изменение структуры базы данных.
- View: обработчик HTTP-запроса.
- URLconf: маршрутизация запросов к view.
