# Разработка макетов
Необходимо разработать макеты основных страниц веб-сайта. Макеты должны отражать структуру страниц, расположение функциональных блоков и навигационные переходы между страницами.

## Требования к макетам

Макеты выполняются в формате wireframe.
Допускается использование любых инструментов проектирования (Figma, Miro, Draw.io, Balsamiq, Pencil Project и др.).

**На макетах должны быть отображены:**
    - шапка сайта (Header);
    - меню навигации;
    - основные функциональные блоки страницы;
    - формы ввода данных (если предусмотрены);
    - кнопки и элементы управления;
    - подвал сайта (Footer);
    - ссылки и переходы между страницами.
**Не требуется:**
    - проработка визуального дизайна;
    - подбор цветовой схемы;
    - использование изображений;
    - оформление текста и типографики;
    - адаптивная верстка.

Все элементы должны быть представлены в виде условных блоков и подписей.

## Минимальный набор страниц

    - Главная страница.
    - Страница каталога/списка объектов.
    - Страница детального просмотра объекта.
    - Страница авторизации или регистрации.
    - Страница обратной связи (или личного кабинета).

## Дополнительное требование

Необходимо показать пользовательские сценарии переходов между страницами:

   - переход с главной страницы в каталог;
   - переход из каталога на страницу объекта;
   - переход на страницу авторизации;
   - возврат на главную страницу через навигацию.

Для обозначения переходов рекомендуется использовать стрелки и подписи.

**Wireframe (вайрфрейм)** — это схематичный макет интерфейса, который показывает структуру страницы, расположение элементов и логику взаимодействия пользователя с системой без проработки дизайна.

Простыми словами, wireframe — это «черновик» страницы сайта или приложения.
